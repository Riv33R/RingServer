# RingScheduler 🔔

**RingScheduler** — это легковесная и удобная система оповещений ("Школьные звонки"), разработанная на Python (Flask) с использованием фонового планировщика APScheduler.
Программа предназначена для работы под ОС Windows и умеет управлять расписанием звуковых звонков, а также работает как музыкальный плеер для оператора.

![Главная панель](https://via.placeholder.com/800x400.png?text=RingScheduler+Dashboard) *(скриншот панели)*

## ✨ Основные возможности

* **Гибкие профили расписаний:** Создавайте различные сценарии звонков (например, "Обычный день", "Сокращенный день", "Суббота", "Экзамен") и привязывайте их к дням недели.
* **Календарные исключения:** Задавайте конкретные профили на определенные даты календаря (перенос праздников, каникулы и т.д.).
* **Интерактивный дашборд:** Живой таймер обратного отсчета до следующего звонка, текущее время и статус системы в режиме реального времени.
* **Ручное управление (Музыкальный плеер оператора):** Отдельная панель для загрузки своих треков, воспроизведения фоновой музыки и изменения системной громкости Windows прямиком из браузера без доступа к рабочему столу сервера.
* **Аудиодвижки:** Поддержка проигрывания `MP3`, `WAV`, `OGG` форматов. Для работы с аудио используются `pygame`, `winsound` и `playsound3`. 
* **Импорт/Экспорт данных:** Удобная загрузка и бэкап расписания звонков в формате CSV.
* **Разделение прав доступа:** Панель настроек защищена. Администратор может менять системные настройки приложения, а Операторы (сотрудники) имеют доступ только к расписанию и музыкальному плееру.

## 🚀 Установка и запуск

Проект написан с расчетом на работу в операционных системах **Windows**, так как для изменения системной громкости используются Windows COM-интерфейсы (`pycaw` и `comtypes`).

### 1. Системные требования
* **Python** версии 3.10 или выше.
* Операционная система Windows (рекомендуется Windows 10/11 или Windows Server).

### 2. Клонирование и зависимости
Склонируйте репозиторий и перейдите в папку с проектом:
```bash
git clone <URL_репозитория>
cd ringsheduler
```

Рекомендуется создать виртуальное окружение, а затем установить зависимости:
```bash
python -m venv venv
venv\Scripts\activate
pip install -r requirements.txt
```

*(Основные используемые библиотеки: `flask`, `APScheduler`, `pygame`, `pycaw`, `comtypes`, `werkzeug`, `playsound3`)*

### 3. Первый запуск
Чтобы запустить сервер:
```bash
python app.py
```
При первом запуске автоматически будет создана база данных SQLite (`scheduler.db`) со стандартным набором тестовых данных (расписание звонков на 7 уроков).

Перейдите в браузере по адресу: **http://127.0.0.1:5000** 

### 4. Данные для входа по умолчанию:
- **Логин:** `admin`
- **Пароль:** `admin`

*(Настоятельно рекомендуется сменить пароль администратора после первого входа!)*

## 📂 Структура проекта
* `app.py` — Главный файл приложения (Flask-сервер, определение маршрутов (роутов) и API).
* `database.py` — Управление базой данных SQLite (профили, настройки, логи вызовов).
* `scheduler.py` — Движок на базе APScheduler, который раз в минуту проверяет БД и отдает команду звонить.
* `audio.py` / `music_player.py` — Системы работы со звуком. Воспроизведение звонков и музыки, а также управление системным миксером Windows.
* `auth.py` — Система авторизации и разграничения доступов пользователей.
* `templates/` — HTML-шаблоны приложения (Jinja2).
* `static/` — CSS стили и JavaScript файлы дашборда.

## 🛠 О проекте
Стек технологий: **Python, Flask, SQLite, HTML5, Vanilla CSS, JS.** 
БД не требует отдельного сервера, все хранится локально в файле `scheduler.db`. Все аудио загружаются в локальную папку сервера (по умолчанию `uploads/`).
