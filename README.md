# RingScheduler — Система школьных звонков

Веб-приложение для автоматической подачи школьных звонков.  
Стек: **Python 3.8+ + Flask + APScheduler + SQLite**  
Разворачивается на **Windows Server 2012** как служба Windows (через NSSM).

---

## Быстрый старт

```
http://localhost:5000
Логин:  admin
Пароль: admin123
```

> ⚠️ **Смените пароль** в Настройках сразу после первого входа!

## Установка

### Шаг 1. Установите Python 3.8+

Скачайте установщик с [python.org/downloads](https://python.org/downloads/windows/).  
При установке отметьте галочку **"Add Python to PATH"**.

Проверьте установку:
```cmd
python --version
```

### Шаг 2. Скопируйте файлы приложения

Скопируйте всю папку `ringsheduler` на сервер, например:
```
C:\ringsheduler\
```

### Шаг 3. Создайте виртуальное окружение и установите зависимости

```cmd
cd C:\ringsheduler
python -m venv venv
venv\Scripts\activate
pip install -r requirements.txt
```

> Если нет интернета на сервере, установите зависимости на другой машине и скопируйте 
> папку `venv` целиком, или создайте wheel-пакеты с помощью `pip download`.

### Шаг 4. Тестовый запуск

```cmd
start.bat
```

Откройте браузер: [http://localhost:5000](http://localhost:5000)

## Установка как Windows Service (Production)

### Скачайте NSSM

1. Скачайте NSSM: [nssm.cc/download](https://nssm.cc/download)
2. Скопируйте `nssm.exe` из архива (папка `win32` или `win64`) в папку `C:\ringsheduler\`

### Установите службу

Запустите **от имени Администратора**:
```cmd
install_service.bat
```

### Управление службой

```cmd
nssm start RingScheduler    # Запустить
nssm stop RingScheduler     # Остановить
nssm restart RingScheduler  # Перезапустить
nssm remove RingScheduler   # Удалить
```

Или через графический интерфейс: **services.msc** → "RingScheduler - Школьные звонки"

## Настройка IIS (опционально)

Если хотите, чтобы приложение было доступно на стандартном порту 80:

1. Установите модуль **Application Request Routing (ARR)** и **URL Rewrite** в IIS
2. Создайте новый сайт или используйте Default Web Site
3. Добавьте правило обратного прокси (Reverse Proxy):
   - Входящий URL: `/*`  
   - Исходящий URL: `http://localhost:5000/{R:0}`
4. В файле `web.config` в корне IIS-сайта:

```xml
<?xml version="1.0" encoding="UTF-8"?>
<configuration>
  <system.webServer>
    <rewrite>
      <rules>
        <rule name="RingScheduler Proxy" stopProcessing="true">
          <match url="(.*)" />
          <action type="Rewrite" url="http://localhost:5000/{R:1}" />
        </rule>
      </rules>
    </rewrite>
  </system.webServer>
</configuration>
```

## Настройка аудио

### Режим 1: Воспроизведение MP3/WAV (по умолчанию)

1. Перейдите в **Настройки** → раздел "Загрузка аудиофайлов"
2. Загрузите файл звонка (.mp3, .wav, .ogg)
3. В поле "Аудиофайл по умолчанию" — выберите загруженный файл
4. Установите режим работы: **Воспроизведение аудио**

> ⚠️ Аудио воспроизводится *на сервере*. Убедитесь, что к серверу подключены колонки
> или что аудиовыход маршрутизирован на нужное устройство.

### Режим 2: Управление физическим реле

1. Адаптируйте файл `relay_example.bat` под ваше оборудование (см. комментарии внутри)
2. В **Настройках** укажите путь к скрипту реле
3. Установите режим: **Запуск скрипта (реле)**

#### Поддерживаемые типы реле:

| Способ | Описание |
|---|---|
| USB-реле | `hidusb-relay-cmd.exe on/off` |
| HTTP API | Shelly 1, Sonoff, умные реле |
| COM-порт | RS-232 реле через `mode COM1 dtr=on/off` |
| GPIO+утилита | Для Raspberry Pi (запуск через SSH) |

## Конфигурация (config.json)

Файл создаётся автоматически при первом запуске. Параметры:

| Ключ | По умолчанию | Описание |
|---|---|---|
| `PORT` | `5000` | Порт HTTP-сервера |
| `HOST` | `0.0.0.0` | Сетевой интерфейс |
| `BELL_MODE` | `"audio"` | Режим: `audio`, `script`, `both`, `log` |
| `BELL_DURATION` | `5` | Длительность звонка (секунды) |
| `DEFAULT_SOUND` | `""` | Путь к аудиофайлу по умолчанию |
| `RELAY_SCRIPT` | `""` | Путь к скрипту реле |
| `DEBUG` | `false` | Режим отладки Flask |

## Возможные проблемы

**Проблема:** `pip install` не работает без интернета  
**Решение:** Скачайте `.whl`-файлы заранее: `pip download -d wheels -r requirements.txt`  
На сервере: `pip install --no-index --find-links=wheels -r requirements.txt`

**Проблема:** Звук не воспроизводится на сервере  
**Решение:** Убедитесь, что аудио устройство активно. Попробуйте режим `log` для тестирования без аудио.

**Проблема:** Служба не запускается  
**Решение:** Проверьте `logs\service_stderr.log`. Убедитесь, что путь к Python в `install_service.bat` указан верно.

**Проблема:** Порт 5000 занят  
**Решение:** Измените `PORT` в `config.json` на другой (например, 8080).
